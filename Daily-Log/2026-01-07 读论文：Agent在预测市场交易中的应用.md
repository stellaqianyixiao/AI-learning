
## 1. 核心痛点：预测市场的“碎片化”

目前的预测市场（如 Polymarket）存在严重的**碎片化**问题 。

* 
**语义重复**：有些合约描述几乎一样（例如：“X 会在 11 月发生吗？” vs “X 会在 11 月 30 日前发生吗？”） 。


* 
**逻辑关联**：不同合约间存在因果、蕴含或排他关系，但平台标签往往太粗糙，难以支持深度挖掘 。


* 
**统计局限**：传统的量化方法需要大量历史价格数据，而预测市场合约通常只结算一次，缺乏足够的长序列数据 。

## 2. 产品方案：基于 Agentics 的智能体流水线

作者设计了一套端到端的 AI 智能体工作流，其核心框架是 **Agentics** 。这个框架对产品经理很有启发，因为它提倡**数据导向的编程模型**：

### A. 核心技术组件：MCP 工具 

该流水线利用 **Model Context Protocol (MCP)** 工具，将复杂任务拆解为三个标准化阶段：

1. 
**聚类 MCP (Clustering)**：利用向量空间模型，根据合约文本的语义距离，将数以百计的市场自动分组成若干话题簇 。


2. 
**标签 MCP (Cluster Labeling)**：使用 LLM 为每个簇分配分类标签（如：政治、加密、体育等），确保数据可审计且分类一致 。


3. 
**关系发现 MCP (Relationship Discovery)**：这是最关键的一步。智能体在同一聚类内寻找**成对关系**，预测它们是“同向（Same-outcome）”还是“反向（Different-outcome）”解析 。



### B. “逻辑转换” (Transduction) 的设计哲学 

智能体被视为一个**状态转换器**：

* 
**输入**：Typed Objects（带有特定 Schema 的市场对象，如 `SingleMarket`） 。


* 
**输出**：符合特定 Schema 的新对象（如 `MarketRelation`） 。



> 
> **PM 视角**：这种基于 Schema 的约束生成（Schema-constrained generation）能显著降低幻觉，确保输出结果可以直接被交易系统或数据库消费 。

## 3. 实证结果：AI 真的能赚钱吗？

研究团队使用 2025 年 4 月至 7 月的 Polymarket 真实历史数据进行了回测 。

* 
**预测准确率**：AI 识别市场间关系的准确率稳定在 **60%-70%** 之间 。


* 
**交易回报**：基于 AI 发现的关系，实施简单的“领头羊-追随者”（Leader-Follower）策略——即当关系对中的一个合约结算后，立即交易另一个合约 。


* 
**平均回报率 (ROI)**：在 4 月至 6 月期间表现优异，平均 ROI 约 **20%**，部分月份（6月）中位数回报甚至达到 **45.7%** 。


* 
**风险警示**：7 月份出现亏损，说明 AI 表现受市场底层分布和语义清晰度的影响 。

## 4. 给 AI 产品经理的启示

对AI效率工具/金融AI产品设计的启示：

* 
**从“对话”转向“流”**：不要只做一个聊天机器人。要把复杂的业务逻辑拆解为多个 **MCP 工具**，通过“聚类 -> 标注 -> 推理”的流式处理来解决专业领域问题 。


* 
**结构化是第一生产力**：定义好 `AType`（Typed Schemas）。让 LLM 处理结构化转换，而不是生成一段散文。这样才能实现“AI 发现信号 -> 程序化执行”的闭环 。


* 
**自信度过滤（Confidence Score）**：在输出中强制要求 LLM 提供 `confidence_score` 。在回测中，只处理置信度  的样本，这对于构建可信的金融级 AI 必不可少 。


* 
**解决碎片化的价值**：在信息过载的领域（如电商、金融、法律），利用 AI 建立“关联图谱”本身就是一种极具商业价值的产品能力 。
